# Play release checklist

## Engineering

- [ ] Complete Phases 1–5 and all release acceptance criteria.
- [ ] Run unit, lint, debug, instrumented, minified release, and bundle tasks.
- [ ] Validate AAB, 16 KB compatibility, Baseline Profile, and Macrobenchmark evidence.
- [ ] Audit final merged manifest, permissions, SDK inventory, and dependency verification.
- [ ] Test API 24, API 37, reference performance device, tablets, foldable, RTL, and 200% font.
- [ ] Provide production signing only outside version control and enable Play App Signing.

## Policy and listing

- [ ] Host an HTTPS, no-login privacy policy and link it in-app and in Play Console.
- [ ] Complete Data Safety from the final SDK/manifest behaviour.
- [ ] Confirm account creation: No; app access: unrestricted core flow.
- [ ] Decide truthful 13+ target audience and complete content rating.
- [ ] Complete Ads/AD_ID/CMP declarations only if ads are enabled in Phase 5.
- [ ] Verify the current closed-test gate in the developer’s Play Console.
- [ ] Run internal testing and Play pre-launch report.
- [ ] Use a staged rollout and monitor crashes, ANRs, reviews, and processing failures.

Android developer verification begins rolling out in September 2026 and must be completed for the publishing account as applicable.
